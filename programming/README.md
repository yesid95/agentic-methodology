# Programming Workflows

The programming domain will define structured workflows for human-led software development with specialized AI assistance.

## Purpose

This domain focuses on coding, review, testing, refactoring, and implementation documentation while preserving human control over scope, validation, commits, and repository direction.

## Expected Future Roles

- Implementation support
- Code review support
- Test and validation support
- Documentation support

## Intended Workflows

- Small scoped implementation iterations
- Manual review before commits
- Documentation updates aligned with code changes
- Controlled context sharing between responsibilities

## Current Status

Foundational structure only. Roles, protocols, prompts, and templates have not yet been defined.
